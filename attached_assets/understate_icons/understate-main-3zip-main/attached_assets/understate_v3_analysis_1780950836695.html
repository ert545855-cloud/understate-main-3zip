Kritik hatalar — deploy'u engeller
KRİTİK
server.js vs server/main.js çakışması
Kök dizindeki server.js eski, hiçbir route/socket'ı bağlamıyor; Socket.IO sadece boş game:action dinliyor. package.json server/main.js'i işaret ediyor — doğru. Ama server.js silinmezse karışıklık ve potansiyel deploy hatası oluşur.
KRİTİK
SQL şeması belirsizliği — 5 farklı dosya var
sql/schema_full.sql tabloya players diyor; server/migrations/schema.sql users diyor. dbService.js ve tüm route'lar users kullanıyor. schema_full.sql'ü çalıştırırsan app çöker. Hangi dosyanın Supabase/RDS'e uygulanacağı belirtilmemiş.
KRİTİK
mongoose dependency yok ama 3 model dosyası var
server/models/User.js, Room.js, ChatMessage.js hepsi mongoose kullanıyor. Ancak package.json'da mongoose yok. Bu dosyalar hiçbir yerde require edilmiyor — ölü kod. npm install sonrası require('mongoose') çağrısı olursa uygulama anında crash eder.
KRİTİK
GamesScreen.js ve SocialScreen.js tamamen boş
Her iki dosya sadece 1 satır yorum içeriyor; asıl içerik yoktur deniyor, alt sayfalara bölünmüş. app.jsx'te bu ekranlara navigasyon varsa runtime'da beyaz ekran verir.
KRİTİK
Çoğu frontend verisi localStorage'da, sunucuya gitmiyor
app.jsx'te S.load/S.save (localStorage) 511 kez kullanılıyor, socket.emit/fetch yalnızca 51 kez. Oyun verileri (envanter, işler, gang, PvP, madencilik, casusluk) sunucuya kaydedilmiyor. Kullanıcı telefonu değiştirirse tüm ilerleme kaybolur. Multiplayer mantığı çalışmaz.
Eksik özellikler — yarım kalmış
EKSİK
Firebase/Supabase kalıntıları temizlenmemiş
index.html'de Firebase yorum satırları var. render.yaml'da SUPABASE_URL, SUPABASE_SERVICE_KEY, MONGODB_URI hâlâ env var olarak tanımlı. .env'de MONGODB_URI= boş satır duruyor. Bu karışıklık yanlış config uygulanmasına yol açar.
EKSİK
SMTP, VAPID, GIPHY, AWS S3 ayarları boş
.env'de SMTP_HOST, VAPID_PUBLIC_KEY, GIPHY_API_KEY, AWS_ACCESS_KEY_ID, AWS_S3_BUCKET boş. E-posta bildirimleri, web push, GIF arama ve dosya yükleme hiçbiri çalışmaz.
EKSİK
SpyPage, MiningPage, PvpPage — localStorage kullanıyor, server yok
Bu üç sayfa useLs (localStorage) ile çalışıyor. Sunucu tarafında karşılık gelen endpoint'ler ya yok ya da bağlı değil. Veriler tek cihaza bağlı kalır.
EKSİK
CasinoPage server entegrasyonu yok
406 satır UI var ama kazanç/kayıp sunucu tarafında doğrulanmıyor. Anti-cheat çalışmaz, para manipülasyonu mümkün.
EKSİK
NpcPlayersPage gerçek NPC yapay zekası yok
Sadece statik veri gösteriyor; NPC davranışı, ekonomik etki veya oyunla etkileşim yok.
EKSİK
iOS kurulumu eksik
package.json'da ios:add scripti var ama android/ klasörü mevcutken ios/ klasörü hiç yok. iOS dağıtımı sıfırdan yapılmalı.
EKSİK
Rate limiter in-memory — sunucu yeniden başlayınca sıfırlanıyor
rateLimiter.js bir Map kullanıyor. AWS EB'de sunucu restart edince tüm limitler sıfırlanır. Birden fazla instance varsa sınır aşılabilir. Redis veya DB destekli limiter gerekli.
EKSİK
Babel ile runtime JSX derleme — production için uygun değil
index.html CDN'den Babel yükleyip JSX'i tarayıcıda compile ediyor. Bu development çözümü; production'da ciddi performans sorunu, SEO sorunu ve yavaş yükleme yaratır. Webpack/Vite build adımı gerekli.
Uyarılar — çalışır ama düzeltilmeli
UYARI
render.yaml AWS deploy için yanıltıcı
Render.com için yazılmış config dosyası AWS paketinde duruyor. PORT 10000 (Render) vs 8080 (EB). Birini kullanırken diğerini düzeltmeyi unutmak yaygın hata kaynağı.
UYARI
EB WebSocket konfigürasyonu eksik adım içeriyor
AWS_DEPLOY.md'de Load Balancer Stickiness etkinleştirmesi manuel adım olarak belirtilmiş ama .ebextensions'a eklenmemiş. Unutulursa Socket.IO polling çalışmaz.
UYARI
ALLOWED_ORIGINS production'da wildcard (*) olmamalı
.env'de varsayılan ALLOWED_ORIGINS=*. Production'da kesin domain belirtilmeli, yoksa güvenlik açığı oluşur.
UYARI
js/ klasöründe eski Firebase dosyaları duruyor
firebase-init.js, firebase-init-v5.js, firebase-init-v6.js, firebase-save.js vb. yükleniyor mu belirsiz. Paket boyutunu şişiriyor, karışıklık yaratıyor.
UYARI
node_modules yok — EB'de npm install çalışmalı
Doğru yaklaşım bu, ama .npmrc veya bağımlılık lock dosyası (package-lock.json) eksikse farklı versiyonlar kurulabilir.
